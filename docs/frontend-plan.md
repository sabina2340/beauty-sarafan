# Каркас плана разработки фронтенда

Ниже — последовательный план запуска фронтенда с приоритетом на **быстрый выход в полезную публичную версию**.

## Этап 1 — Публичная часть (для клиентов) ✅ первая цель

Цель этапа: сделать рабочую «витрину» без регистрации, чтобы пользователи уже могли находить мастеров и смотреть информацию.

### 1. Базовая структура приложения
- Поднять каркас фронта (layout, навигация, footer, базовые стили).
- Настроить роутинг и страницы:
  - `/` (главная)
  - `/masters` (каталог мастеров)
  - `/masters/[id]` (карточка мастера)
- Добавить общие состояния UI: loading / empty / error.

### 2. Главная страница (`/`)
- Добавить ключевой CTA-блок.
- Основная кнопка: **«Найти мастера»** с переходом на `/masters`.
- (Опционально) показать 3–6 популярных мастеров/категорий как быстрые переходы.

### 3. Каталог мастеров (`/masters`)
Реализовать выдачу мастеров и фильтрацию:
- Фильтр по категории (`slug`).
- Фильтр по городу.
- Поиск по строке (`q`).
- Синхронизация фильтров с query-параметрами URL (чтобы можно было делиться ссылкой).

#### Что важно по UX
- Фильтры должны обновлять список без перезагрузки страницы.
- При пустой выдаче показывать понятное сообщение и быстрый сброс фильтров.
- Сохранять текущие фильтры при переходе в карточку мастера и возврате назад.

### 4. Карточки мастеров (в каталоге)
Для каждой карточки показать минимум:
- Имя/бренд мастера.
- Категорию/направление.
- Город.
- Короткое описание (1–2 строки).
- Кнопку/ссылку «Подробнее» → `/masters/[id]`.

### 5. Страница мастера (`/masters/[id]`)
Обязательные блоки:
- Описание мастера.
- Услуги.
- Контакты.

Опциональный блок:
- Объявления мастера (если используется маршрут `/masters/{id}/ads`).

### 6. API-интеграция и данные
- Описать клиент для API (типизация, базовые DTO, обработка ошибок).
- Подключить эндпоинты:
  - список мастеров;
  - детальная информация о мастере;
  - (опционально) объявления мастера.
- Предусмотреть скелетоны/прелоадеры на время загрузки.

### 7. Качество и готовность этапа 1
- Проверить адаптивность (mobile/tablet/desktop).
- Проверить базовую доступность (контраст, focus-state, aria-label для интерактивных элементов).
- Добавить SEO-минимум: title, description, open graph для `/` и `/masters/[id]`.
- Прогнать smoke-тест пользовательского сценария:
  - открыть `/`;
  - нажать «Найти мастера»;
  - применить фильтры;
  - открыть карточку мастера;
  - увидеть описание, услуги и контакты.

---

## Что получаем после этапа 1
- Рабочую публичную витрину, доступную **без регистрации**.
- Базовую воронку: `Главная → Каталог → Карточка мастера`.
- Основание для следующего этапа (авторизация, кабинет клиента, отклики/запись).

---

## Рекомендуемое продолжение после этапа 1

### Этап 2 — Авторизация и кабинет клиента
- Регистрация/вход.
- Профиль клиента.
- Избранные мастера.

### Этап 3 — Механика заявок/записи
- Отправка заявки мастеру.
- История заявок.
- Статусы и уведомления.

### Этап 4 — Кабинет мастера
- Редактирование профиля и услуг.
- Управление объявлениями.
- Обработка входящих заявок.

### Этап 5 — Улучшения
- Рейтинг/отзывы.
- Оплата (если нужна).
- Продвинутая аналитика и A/B гипотезы.
